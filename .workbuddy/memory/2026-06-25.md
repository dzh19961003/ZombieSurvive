# 2026-06-25 工作日志

## JsonLoader 静态工具类创建

在 `Scripts/Tools/JsonLoader.cs` 新建 JSON 配置加载工具类，用于从 res:// 全局检索 JSON 文件并反序列化为 List<T>。

### 设计要点
- 命名空间 `MyProject.Tools`，静态类 `JsonLoader`
- 核心方法：`List<T> LoadToList<T>(string jsonName)`
  - 输入 JSON 文件名（可带可不带 .json 扩展名）
  - 在 res:// 下递归搜索匹配文件名的 .json 文件（全局检索，不限定 Config 目录）
  - 通过 Godot.FileAccess 读取文件内容（兼容导出版本）
  - 用 System.Text.Json 反序列化，PropertyNameCaseInsensitive=true 兼容 camelCase JSON ↔ PascalCase 属性
  - 失败时返回 null 并通过 GD.PrintErr 输出错误信息
- 搜索时跳过 .godot 等隐藏目录
- JsonSerializerOptions 设为静态单例避免重复分配

### 编译问题修复
- Godot.FileAccess 与 System.IO.FileAccess 命名冲突 → 移除 System.IO using，用全限定 Godot.FileAccess
- 项目未启用 nullable，去掉所有 `?` nullable 注解

### 运行时搜索失败修复（res:// 路径拼接坑）
**问题**：`DirAccess.Open("res:/Config")` 返回 NULL，无法打开子目录。
**根因**：`path.TrimEnd('/') + "/" + entry` 对 `res://` 路径生效时，`TrimEnd('/')` 把双斜杠全部去掉，`res://` 变成 `res:`，拼接后只有 `res:/Config`（单斜杠），Godot 无法识别。
**修复**：添加 `CombinePath` 方法，对 `res://` 根目录特殊处理（`"res://" + entry`），其他路径正常拼接。

### DirAccess 嵌套列举坑（已修复）
初版在 ListDirBegin 期间嵌套 DirAccess.Open 打开子目录，导致枚举被中断。改为两阶段：先收集所有条目，ListDirEnd 后再递归。

### LoadToDic 方法
新增 `Dictionary<int, T> LoadToDic<T>(string jsonName)`：
- 先反序列化为 List<T>，再通过反射读取 ID 属性转为 Dictionary
- ID 属性用 `Convert.ToInt32()` 容错，兼容 double/long/short 等数值类型
- ID 为 null 或无法转 int → 警告并跳过该条；类无 ID 属性 → 报错返回 null

### PlaceInfo 字段类型修正
JsonToClassGenerator 插件默认把所有数字生成为 double，已手动将 PlaceInfo 中所有整数字段改为 int（ID, difficulty, weight, stage, eventNum, enterEventID, endEventID, item）
