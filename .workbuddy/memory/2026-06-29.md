## 2026-06-29

### 新增存档系统（SaveManager + ISaveable 接口）

- 创建 `Scripts/Tools/ISaveable.cs` — 存档接口，定义 `SaveKey`、`GetSaveData()`、`LoadSaveData()`
- 创建 `Scripts/Tools/SaveManager.cs` — 单存档管理器单例，挂 Main.tscn
  - 接口式设计：各脚本自行实现 ISaveable 并 Register，SaveManager 只管遍历和 JSON 读写
  - 单存档路径 `user://save.json`
  - 提供 Save() / Load() / HasSaveFile() / DeleteSaveFile()
  - 存档结构：`{ "meta": { "version":1, "save_time":"..." }, "data": { "player":{...}, "level":{...} } }`
- 创建 `Scripts/GameLogic/PlayerManager.cs` — 使用示例，展示 ISaveable 的完整实现模式

### 改造：用 Godot 分组替代手动注册

- 移除了 SaveManager 的 Register/Unregister 方法
- Save/Load 时通过 `GetTree().GetNodesInGroup("Persist")` 自动发现存档节点
- 各脚本只需在 `_Ready()` 里 `AddToGroup("Persist")` 即可参与存档
- SaveKey 改用 `GetPath()` 自动生成（如 `/root/Main/PlayerManager`），无需手动起名
- LoadSaveData 示例改用 `GetValueOrDefault` 简化写法，一行一个字段
  - 修正：Godot Dictionary 没有 GetValueOrDefault，改用 `ContainsKey` + 三元表达式
