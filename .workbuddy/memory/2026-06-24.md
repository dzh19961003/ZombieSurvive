# 2026-06-24 工作日志

## JsonToClassGenerator Godot 插件开发

为 Godot 4.7 + C# 项目开发了一个编辑器插件，用于从 JSON 配置文件自动生成 C# 数据类。

### 插件结构
- `addons/JsonToClassGenerator/plugin.cfg` — 插件元数据
- `addons/JsonToClassGenerator/JsonToClassGenerator.gd` — 插件主脚本（GDScript）
- `addons/JsonToClassGenerator/JsonLoader.cs` — JSON 加载工具类源文件（随插件分发，点击生成时自动复制到 Scripts/Tools/）

### 功能
- 在 Godot 编辑器底部面板添加 "JSON→C#" 标签页
- 提供文件夹路径输入栏（默认 "Config"）和「浏览...」按钮
- 点击「生成」按钮，自动读取指定文件夹中的所有 JSON 文件
- 根据 JSON 文件名生成 PascalCase 类名（如 place_info → PlaceInfo）
- 根据 JSON 字段值自动推断 C# 类型（int/string/bool/List<T> 等）
- 生成的 .cs 文件输出到 `Scripts/GeneratedClasses/` 目录

### 已在 project.godot 中启用插件
`[editor_plugins]` 段已添加 `enabled=PackedStringArray("res://addons/JsonToClassGenerator/plugin.cfg")`

### 示例输出
用 `Config/place_info.json` 测试生成 `Scripts/GeneratedClasses/PlaceInfo.cs`，字段类型推断正确。

### 插件更新（ConfigManager 单例生成）
- 点击「生成」时除了生成类文件，还会自动生成 `Scripts/Tools/ConfigManager.cs` 单例脚本
- ConfigManager 继承 Node，提供 `Instance` 静态单例
- 为每个配置表类生成 `xxxList`（camelCase + List）和 `xxxDic` 属性
- `_Ready()` 中通过 `JsonLoader.LoadToList<T>` 和 `JsonLoader.LoadToDic<T>` 加载数据
- 命名规则：类名 PascalCase（PlaceInfo），属性 camelCase（placeInfoList/placeInfoDic），JSON 文件名 snake_case（place_info）

### 已知问题修复
- GDScript 中 `class_name` 是保留关键字，不能用作变量名 → 改为 `cs_class_name`
- Godot 4 中 `Label.VERTICAL_ALIGNMENT_CENTER` 不存在 → 直接用整数值 `1`
- Godot C# 脚本需要命名空间，否则报 "Missing class qualified name" → 所有生成的 .cs 文件都包裹在 `namespace MyProject` 中
- JsonLoader.cs 现在作为插件的一部分，存放在 `addons/JsonToClassGenerator/JsonLoader.cs`，点击「生成」时自动复制到 `Scripts/Tools/JsonLoader.cs`
- 数字类型推断优化：Godot JSON 解析器将所有数字转为 float，现通过正则匹配原始 JSON 文本精确判断 int/double。不含小数点的数字 → int，含小数点 → double
